## 2026-07-13：OVO 八卡分片

- 将 OVO 评测改为 Accelerate 多进程：每个 rank 使用独立模型和独立样本分片，写出 `veristream_rank_*.json`，rank 0 合并并计分。
- 显式设置 `apply_padding=False`，防止样本数少于 GPU 数量时 Accelerate 复制样本造成重复评测。
- 正式全量命令使用独立结果目录，避免与修复前的 smoke shard 混淆。

## 2026-07-14：感知-推理解耦原型

- 新增 `lib/veristream_dual_role.py`：视频级 L0/L1/L2 证据索引、问题级 L3 工作记忆、混合检索、候选/验证/隔离/淘汰状态转换，以及安全 JSON 工具循环。
- 新增工具：`search_memory`、`inspect_segment`、`verify_evidence`、`compare_segments`、`answer`。感知服务只生成观察，推理服务只决定动作和答案；内存写入与工具权限由本地编排器掌控。
- 新增 `build_veristream_evidence_index.py` 和 `query_veristream_dual_role.py`，支持本地 Qwen3-VL 或 OpenAI 兼容 vLLM 服务。
- 当前 Conda 环境未安装 vLLM；部署命令已写入复现指南，待全量 OVO 任务结束后安装并进行服务可用性/显存基准测试。
- 新增 `eval_qwen3vl_ovo_dual_role.py`：复用 Accelerate 和 OVO 因果切分，只跑 Backward/Realtime；每个 rank 支持双模型或 `--share-model`，每样本增量写入 JSONL checkpoint。
