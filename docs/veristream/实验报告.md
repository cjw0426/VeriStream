# VeriStream：因果证据记忆与感知-推理解耦的长视频理解智能体

## 摘要

长视频理解的核心困难不是简单地增加输入帧数，而是在有限视觉上下文中同时保留长期历史和
当前状态。本文在 training-free 设定下研究这一问题。我们首先在 OVO-Bench 上比较 recent
窗口、均匀抽帧、CLIP 检索和文本记忆，发现增加 recent 帧数对 Backward 的帮助有限，同时会
显著损害 Real-time；而 `Recent4 + text memory` 能在保留当前视觉输入的同时补充历史信息。
基于这一观察，我们设计 VeriStream：感知角色建立带原始时间指针的证据 memory，推理角色
检索并核验候选证据，最终 Hybrid-4 推理同时接收 VERIFIED 历史 memory 和因果最近 4 帧。

在 Qwen3-VL-8B 上，Hybrid-4 的 Backward 为 57.60%，Real-time 为 79.18%，两项平均为
68.39%，相比 text-only 双角色的 55.05%/44.91% 显著恢复 Real-time，并略高于 recent4
baseline 的 Backward/Real-time 两项平均。结果说明历史 memory 和当前视觉上下文需要协同，
完全依赖文本摘要会造成不可逆的视觉信息损失。

## 1. 任务与研究问题

给定视频 $V=(f_1,\ldots,f_T)$、问题 $q$ 和问题可见时刻 $t_q$，系统只能访问因果前缀
$V_{\leq t_q}$。模型单次上下文容量为 $B$，而视频信息量远大于 $B$。系统需要在预算 $K$
内选择工具序列：

\[
\pi=(u_1,\ldots,u_K),\quad
u_k\in\{\texttt{search},\texttt{inspect},\texttt{verify},\texttt{compare},\texttt{answer}\}.
\]

本文关注三个问题：

1. 哪些历史信息应被压缩为持久 memory，哪些信息应在回答时重新观察？
2. 如何让模型主动选择检索、局部重观察和证据核验工具？
3. 感知和推理应共享视觉上下文，还是通过结构化 memory 解耦？

### 2.2 Text memory 预实验

为了降低视觉上下文压力，我们将历史帧压缩为文本，同时保留最近 4 帧。统一流程是：

```text
历史帧选择：uniform16 或 CLIP-topk16
    -> 每 4 帧组成一个视觉组
    -> Qwen3-VL 生成一句英文 caption
    -> 文本模型压缩为最多 3 条 bullet memory
    -> 最终问答输入：最近 4 帧 + 历史文本 memory
```

**Action-fact memory** 的 prompt 关注直接可见动作、对象位置、对象状态和时间事件。对于
“一个人把红杯子放到桌上，然后离开”的视频，它可能生成：

```text
A person places a red cup on the table and walks away.
```

**State-aware memory** 的 prompt 关注过去状态、场景连续性、角色身份和已完成进度，可能生成：

```text
The red cup is left on the table after the placement is completed.
```

前者保留“发生了什么”，后者保留“动作完成后的状态”。设计这两个版本是为了区分两个风险：
事件细节丢失，以及历史动作被误认为当前动作。

2B 结果：

| 方法 | Backward | Real-time | Forward | Total |
| --- | ---: | ---: | ---: | ---: |
| Recent4 + action-fact memory + uniform16 | 56.91 | 73.68 | 38.84 | 56.48 |
| Recent4 + CLIP-topk16 | 52.73 | 69.52 | 43.12 | 55.13 |
| Recent4 + action-fact memory + CLIP-topk16 | 57.30 | 73.59 | 38.74 | 56.54 |
| Recent4 + state-aware memory + uniform16 | 56.34 | 74.32 | 36.09 | 55.58 |
| Recent4 + state-aware memory + CLIP-topk16 | 54.11 | 73.32 | 36.11 | 54.51 |

2B `Recent4 + CLIP-topk16` 的结果来自独立的 CLIP 历史帧检索实验，不包含文本 memory；它在
Backward/Realtime/Forward 上分别为 52.73%/69.52%/43.12%，Total 为 55.13%。结果文件为
`main_experiments/results/ovo_qwen3vl_2b_recent4_clip16_gpu4567/qwen3vl_results_20260721_003655.json`。

8B 结果：

| 方法 | Backward | Real-time | Forward | Total |
| --- | ---: | ---: | ---: | ---: |
| Recent4 + action-fact memory + uniform16 | 62.09 | 79.65 | 39.04 | 60.26 |
| Recent4 + action-fact memory + CLIP-topk16 | 61.93 | 78.36 | 39.56 | 59.95 |
| state-aware memory + uniform16/CLIP-topk16 | TODO | TODO | TODO | TODO |

8B 的 Backward 从 53.92 提升到 62.09，而 Real-time 仅从 81.47 降到 79.65；2B 的收益较弱。
这说明文本 memory 的质量与基座模型的视觉总结能力相关，也支持“memory-perception trade-off”
假设：历史信息压缩可以补充 Backward，但必须保留 recent4 才能维持当前感知。

## 3. VeriStream 方法

### 3.1 总体架构

方法图见 `figures/veristream_overview.svg`，可编辑源图见 `figures/veristream_overview.mmd`。
系统由评测 harness、感知角色、推理角色、证据 memory 和本地工具编排器组成：

```text
因果视频前缀
    -> 感知角色建立 L0/L1/L2 evidence memory
    -> 推理角色 search_memory 获取候选
    -> inspect_segment / verify_evidence / compare_segments
    -> VERIFIED 历史 memory + 最近 4 帧
    -> 最终多模态推理
```

所有模型参数冻结，不使用训练或现成视频 agent 框架。工具执行、memory 状态更新和文件访问
由自主实现的 Python 编排器完成。

### 3.2 分层 Memory

| 层级 | 内容 | 生命周期 |
| --- | --- | --- |
| L0 | 视频路径、chunk 索引、时间区间、原始帧指针 | 原始证据不可随摘要淘汰 |
| L1 | observation、实体、动作、confidence、状态 | 按视频持久化 |
| L2 | 按实体、动作和相邻时间聚合的 event/state | 按视频持久化 |
| L3 | 当前问题检索出的少量 memory ID | 问题结束后清空 |

状态机为：

```text
candidate -> verified
candidate -> quarantined
candidate -> stale/evicted
```

同一视频、重叠 chunk 且文本 Jaccard 相似度较高的观察合并；冲突观察不覆盖，而是保留为
不同版本。`video_id` 命名空间阻止跨视频 memory 污染。

### 3.3 工具调用

推理模型只能输出白名单 JSON：

```text
search_memory(query)
inspect_segment(start_time, end_time, target)
verify_evidence(memory_ids, target)
compare_segments(memory_ids, target)
answer
```

- `search_memory` 使用词元、实体和动作匹配进行可解释召回，并加入 confidence、访问次数和
  VERIFIED 状态奖励。
- `inspect_segment` 根据模型给出的时间区间重新读取局部帧，生成新的 candidate observation。
- `verify_evidence` 根据 L0 指针回取原始帧，要求感知模型输出 `SUPPORTED`、`UNSUPPORTED` 或
  `CONFLICT`。只有 `SUPPORTED` 才升级为 VERIFIED。
- `compare_segments` 使用两个候选时间段的少量原始帧判断状态变化。

工具调用的优势是将昂贵视觉观察集中到问题相关片段，而不是对整个视频无限增加抽帧密度。

### 3.4 感知与推理组织

感知角色问题无关地建立粗粒度全视频索引；推理角色只接收问题、候选摘要和工具结果。旧版
text-only 双角色的最终推理只接收 VERIFIED 文本，导致当前状态、空间关系和 OCR 信息严重丢失。

当前 Hybrid-4 的最终推理上下文为：

```text
历史信息：VERIFIED evidence memory
当前信息：因果最近 4 帧原始视觉
```

这不是退回 recent4 baseline：recent4 没有长期历史 memory，而 Hybrid-4 仍具备持久 memory、
证据来源、局部重观察和核验门控。

## 4. 评测设置

### 4.1 数据集

使用公开 OVO-Bench，包含 1,640 个 Backward/Realtime/Forward 问题：

| Split | 任务类型 | 问题数 |
| --- | --- | ---: |
| Backward | EPM/HLD/ASI | 631 |
| Real-time | STU/OJR/ATR/FPD/ACR/OCR | 837 |
| Forward | REC/SSR/CRR | 172 |

Backward/Realtime 按任务类型计算准确率后取平均；Forward 的 REC、SSR、CRR 按原始 OVO
评分逻辑计算。Hybrid-4 的 Backward/Realtime 使用双角色主入口评测，Forward 使用独立入口补跑；
两者配置一致，因此可以合并报告三类 split 的结果，同时保留原始两 split 结果以便对照。

### 4.2 Baseline 与实现

- 模型：Qwen3-VL-2B/8B-Instruct，冻结参数；
- baseline：同一基座模型、recent/uniform 抽帧、单轮直接问答；
- 解码：正式对照使用 `chunk_duration=1.0`、`fps=1.0`；
- 双角色：`coarse_stride=12`、`max_frames_per_observation=8`、`max_actions=4`；
- Hybrid-4：`final_recent_frames=4`；
- 所有代码注释为中文，实际发送给模型的 prompt 为英文。

## 6. 失败分析与局限

### 6.1 text-only DualRole 失败

旧版 text-only 的 Real-time 只有 44.91%，主要原因是最终推理没有原始视觉输入。感知模型把
多帧压缩为一句 observation 后，小目标、左右关系、OCR 字符和动作顺序无法恢复；严格的
核验门控又使约 18.7% 样本输出 “Evidence is insufficient.”。

### 6.2 Hybrid-4 的剩余问题

1. 最近 4 帧恢复了当前视觉，但可能稀释部分长期历史证据，Hybrid-4 的 HLD 低于 text-only
   DualRole 的 74.73%；
2. ACR 低于 baseline 6.42 个百分点，后续需要 Hybrid-8、局部区域观察或空间关系工具；
3. 每个问题独立建立视频 memory，造成较大运行成本。OVO 中 1,468 个问题只有 512 个不同
   视频，未来应按视频缓存索引并按 $t_q$ 过滤；
4. 当前 `search_memory` 是词元检索，不是 embedding 检索，同义词和表达变体可能召回失败；
5. Hybrid-4 的 Forward 已补充完成，但 Forward 仅使用独立入口运行，后续仍需统一缓存和并行
   策略，以降低与 Backward/Realtime 分开运行带来的成本。

### 6.3 方法边界

Hybrid-4 不是训练方法，也不是现成 agent 框架调用。它的创新点在于 training-free 条件下将
持久、可回溯、可核验的历史 memory 与少量当前原始视觉联合使用，并通过受限工具循环控制
视觉预算。当前结果不支持“所有任务均提升”的结论，支持的是 memory-perception trade-off
和混合上下文设计的机制性结论。

## 8. 借鉴与差异

本文参考 VideoDeepResearch 的主动工具调用思想、VideoLucy 的深层 memory 回溯思想以及
MemDreamer 的感知/推理解耦思想，但没有调用它们的现成视频 agent 框架。我们的差异是：

- 只使用冻结 Qwen3-VL，不进行模型训练；
- 自主实现 JSON 工具白名单、L0-L3 memory 和状态机；
- 每条 observation 保存原始时间/chunk 指针；
- 历史摘要在进入最终答案前必须经过局部原始帧核验；
- Hybrid-4 将 VERIFIED 历史文本和当前最近原始帧联合输入，以显式处理 memory-perception trade-off。

## 9. 结论

预实验表明，简单增加 recent 帧数不能稳定解决长视频理解；文本 memory 能改善长期历史问题，
但纯文本解耦会严重损害当前感知。VeriStream Hybrid-4 用可回溯、可核验的历史 memory 加上
最近 4 帧原始视觉，在 8B OVO-Bench 上将 Real-time 恢复到 79.18%，同时将 HLD 提升到 61.83%。
这支持一种更稳妥的 training-free 设计：历史信息应被结构化、检索和验证，当前状态则保留少量
原始视觉输入。后续重点是按视频缓存 memory、改进语义检索、降低 ACR 空间关系损失，并补齐
Hybrid-4 的 Forward 结果。
