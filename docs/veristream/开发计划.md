## 接口约束

- `EvidenceMemoryStore` 只接受一个视频命名空间，序列化 JSON 必须含 `video_id`。
- `VeriStreamAgent.ingest` 只能接收当前时刻已到达的 chunk；评测器在传入前裁剪时间边界。
- 控制器只能返回 `retrieve`、`inspect_recent`、`verify_and_zoom`、`compare`、`write_or_update`、`answer`。
- 所有实验输出保存配置、结果、trace；原始视频和大日志不提交。
- 双角色服务通过 OpenAI 兼容 HTTP 接口访问；服务端不拥有记忆写权限，所有工具执行和状态转换均由本地编排器控制。

## 验收标准

- 单元测试覆盖命名空间隔离、候选隔离、验证升级、淘汰优先级、JSON 安全和因果摄入。
- StreamingBench 小样本可端到端产生 memory JSON、结果 JSON 和完整工具轨迹。
- 全量比较同时报告准确率、Qwen 调用数/帧数、CLIP 扫描帧数和延迟。
- 双角色消融同时报告索引构建时间、问题平均推理延迟、局部回溯次数、核验拒绝率和记忆缓存复用率。
