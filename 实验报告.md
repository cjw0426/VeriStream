# VeriStream：因果证据记忆与感知-推理解耦的长视频理解智能体

## 摘要

长视频问答的瓶颈并非单纯缺少帧，而是历史证据、当前状态和有限视觉上下文之间的分配不当。
本文在 **training-free** 设定下，以冻结的 Qwen3-VL 为基座，系统研究了原始视觉窗口、文本记忆
和工具化证据记忆的作用。OVO-Bench 预实验显示：扩大 recent window 对 Backward 的收益不稳定，
却持续损害 Realtime；文本记忆可补充历史信息，但纯文本解耦会丢失细粒度视觉状态。基于这一
观察，我们实现 VeriStream：感知角色为视频因果前缀建立带时间指针的分层证据索引，推理角色
按问题检索候选并通过局部重观察和核验门控筛选证据，最终将 `VERIFIED` 历史记忆与最近 4 帧
原始视觉联合输入答案模型。

在 Qwen3-VL-8B 上，VeriStream Hybrid-4 的 Backward/Realtime/Total 分别为 **57.60/79.18/68.39**，
相对于 Recent4 baseline 的 **53.92/81.47/67.70**，Backward 提升 3.68 个百分点，Total 提升
0.69 个百分点，同时避免了 text-only 双角色的 Realtime 崩溃（44.91%）。结果不表明所有任务
都提升，而是验证了“历史用可回溯记忆、当前用少量原始视觉”的机制假设。

## 1. 问题建模

给定视频 $V=(f_1,\ldots,f_T)$、问题 $q$ 以及问题可见时刻 $t_q$，系统只能访问因果前缀
$V_{\le t_q}$。模型上下文预算为 $B$，而 $|V_{\le t_q}|\gg B$。系统需要在工具预算 $K$ 内
选择一系列操作。在预算 $K$ 内，第 $k$ 次操作记为 `u_k`，可选操作集合为
`{search_memory, inspect_segment, verify_evidence, compare_segments, answer}`。工具调用必须满足
因果时间约束，且总次数不超过预算。

评测关注三个机制问题：

1. 哪些历史信息应压缩为持久记忆，哪些信息应在回答时重新读取？
2. 如何让模型主动请求与问题相关的局部视觉证据，而不是无差别增加抽帧密度？
3. 感知和推理共享上下文还是解耦？解耦后的压缩损失如何由原始证据回指和核验弥补？

## 2. 预实验与研究动机

### 2.1 从扩大 Recent 窗口到显式帧检索

我们先复现 SimpleStream 的单轮直接问答基线。该基线只向冻结 Qwen3-VL 提供问题时刻附近的
最近帧，因此可以分别观察“当前感知能力”和“历史信息覆盖能力”。

**Recent 窗口实验。** 已完成的 2B 结果为：

| 方法 | Backward | Realtime | Total (B/R) |
| --- | ---: | ---: | ---: |
| Recent4 | 53.56 | 74.23 | 63.90 |
| Recent16 | 54.19 | 72.95 | 63.57 |

8B 结果为：

| 方法 | Backward | Realtime | Total (B/R) |
| --- | ---: | ---: | ---: |
| Recent4 | 53.92 | 81.47 | 67.70 |
| Recent8 | 55.25 | 79.59 | 67.42 |
| Recent16 | 55.06 | 77.80 | 66.43 |
| Recent32 | 57.35 | 76.57 | 66.96 |
| Recent64 | 56.15 | 75.62 | 65.89 |

扩大窗口没有带来单调收益：8B 从 Recent4 增加到 Recent32，Backward 提升 3.43 个百分点，
但 Realtime 下降 4.90 个百分点，Total 甚至下降 0.74 个百分点；2B 从 Recent4 到 Recent16
也只带来 0.63 个百分点的 Backward 增益，同时 Realtime 下降 1.28 个百分点。我们的解释是
**lost in the middle**：更早的帧确实包含长期证据，但大量相似或无关帧会稀释模型对问题关键帧
的注意力，尤其破坏依赖当前状态的 Realtime 任务。于是问题从“输入更多帧”转化为“如何有选择
地取回历史证据”。

**纯帧检索工具。** 下一步不改变模型，只引入两个轻量的无训练选择器：Uniform 按时间覆盖选择
历史帧，CLIP-TopK 按问题-帧语义相似度选择历史帧。它们的动机分别是覆盖长程事件和集中问题相关
内容。2B 的结果如下：

| 方法 | Backward | Realtime | Total (B/R) |
| --- | ---: | ---: | ---: |
| Uniform32 | 51.70 | 66.49 | 59.10 |
| CLIP-TopK32 | 49.26 | 66.33 | 57.80 |

与 Recent4 相比，两种检索的 Realtime 都明显下降，说明“找到相关历史帧”不能替代对当前帧的
精确感知；CLIP 相似度还可能偏向外观相似而非回答所需的因果证据。该结果促使我们把历史帧和
当前帧同时输入，而不是直接用检索结果替换 Recent 窗口。

**历史帧 + Recent4。** 为保持实时感知，我们将最近 4 帧固定保留，再加入历史 Uniform 或
CLIP-TopK 帧。8B 中 `Recent4 + Uniform28` 和 `Recent4 + CLIP-TopK28` 的 Total 分别为 64.18
和 63.53，均低于 Recent4 的 67.70；2B 中 `Recent4 + CLIP-TopK16` 为 52.73/69.52/61.13，
同样没有带来提升。虽然该设置同时包含历史和当前信息，但原始帧仍以长序列直接拼接，模型需要
在一次上下文中自行区分两种信息，帧数增加带来的 lost-in-the-middle 仍然存在。因此我们需要
一种更紧凑、可被语言模型组织的历史表示。

### 2.2 文本记忆：从“发生了什么”到“现在是什么状态”

为了验证历史信息是否可以从视觉上下文中抽离，我们先把历史帧按 Uniform16 或 CLIP-TopK16
选出，再由冻结 Qwen3-VL 生成英文观察，最后压缩为最多 3 条 bullet memory。答案阶段仍然
输入最近 4 帧，因此该实验单独测量“历史摘要能否补充当前视觉”。我们比较了两种有明确归纳
偏置的记忆：

**Action-fact memory（事件事实记忆）**只保留当前帧组中直接可见、可定位的事实：谁做了什么、
对什么对象做、对象在哪里、动作的先后顺序是什么。它尽量避免因果推断和跨片段补全。例如，
视频中一个人把红杯放到桌上并离开时，记忆写作：

> `A person places a red cup on the table and walks away.`

这种表示适合 Backward 的事件回溯，也容易通过原始时间段重新核验。

**State-aware memory（状态感知记忆）**进一步尝试维护动作完成后的持久状态、实体关系和场景
进度，回答“动作之后现在保持什么”。同一片段可能写作：

> `The red cup remains on the table after the person leaves.`

它对“当前状态”和跨时间关系更紧凑，但需要模型把多个观察拼接成稳定状态；一旦某个动作被漏
检或时间边界不准，状态摘要就可能把推断当成事实，产生错误的持续状态。

**文本记忆对照实验。** 下表中的所有记忆方法都保留最近 4 帧原始视觉；历史帧先由 Uniform16
或 CLIP-TopK16 选择，再由同一基座模型生成对应类型的英文记忆。`Total` 始终为
`(Backward + Realtime) / 2`。

| 模型 | 历史表示 | 历史选择 | Backward | Realtime | Total (B/R) |
| --- | --- | --- | ---: | ---: | ---: |
| Qwen3-VL-2B | 无记忆（Recent4） | - | 53.56 | 74.23 | 63.90 |
| Qwen3-VL-2B | Action-fact | Uniform16 | 56.91 | 73.68 | 65.30 |
| Qwen3-VL-2B | Action-fact | CLIP-TopK16 | 57.30 | 73.59 | 65.45 |
| Qwen3-VL-2B | State-aware | Uniform16 | 56.34 | 74.32 | 65.33 |
| Qwen3-VL-2B | State-aware | CLIP-TopK16 | 54.11 | 73.32 | 63.72 |
| Qwen3-VL-8B | 无记忆（Recent4） | - | 53.92 | 81.47 | 67.70 |
| Qwen3-VL-8B | Action-fact | Uniform16 | 62.09 | 79.65 | 70.87 |
| Qwen3-VL-8B | Action-fact | CLIP-TopK16 | 61.93 | 78.36 | 70.15 |

实验结果支持以下选择逻辑：两种记忆都能改善 2B 的 Backward，但差异很小；在 CLIP 历史选择下，
Action-fact 的 Total 为 65.45，而 State-aware 为 63.72，前者高 1.73 个百分点。Uniform 下二者
分别为 65.30 和 65.33，说明 State-aware 并非没有价值，但收益不稳定。8B 上 Action-fact +
Uniform16 达到 62.09/79.65/70.87，并明显优于 Recent4 的 53.92/81.47/67.70。我们因此将
Action-fact 选为默认记忆类型：它更接近原始观测、时间边界清晰、对不同抽帧策略更稳健，也更
适合后续的证据核验；State-aware 作为可扩展的状态聚合方向保留。在已测试的“简单选择器 +
Recent4”基线中，Action-fact + Uniform16 在 8B 上取得最高的 70.87 Total，说明语言压缩确实
比直接堆叠历史帧更有效；这验证了“历史信息应先组织成紧凑语言先验，再与当前视觉结合”的
猜想，也构成引入 VeriStream 的直接动机。

这一步仍暴露出两个问题。第一，纯文本摘要无法恢复 OCR、左右关系、小目标外观等不可压缩的
视觉细节；第二，摘要没有原始时间指针时，错误记忆难以追责。因此我们没有简单扩大摘要长度，
而是把 Action-fact 的“直接可见事实”升级为带时间 provenance 的结构化 observation，并为每条
观察保留原始 chunk 指针。随后加入 `inspect_segment` 和 `verify_evidence`，让推理角色在需要
时回看局部视频、验证候选事实，最终形成 VeriStream 的 evidence memory。

## 3. VeriStream 方法

### 3.1 总体架构

方法流程为：

```text
因果视频前缀 V<=tq
    -> 感知角色：建立带时间指针的 L0/L1/L2 evidence memory
    -> 推理角色：search_memory 检索候选
    -> inspect_segment / verify_evidence / compare_segments
    -> VERIFIED 历史记忆 + 最近 4 帧原始视觉
    -> 最终推理与答案
```

所有模型参数冻结；工具执行、记忆状态迁移和停止条件由仓库中的 Python harness 自主实现，
没有调用现成视频 agent 框架。论文风格方法图作为提交材料单独提供，代码仓库不上传图片资源。

### 3.2 分层证据记忆

| 层级 | 保存内容 | 作用与生命周期 |
| --- | --- | --- |
| L0 raw pointers | 视频 ID、chunk 路径、起止时间、帧索引 | 不删除，作为可回溯证据源 |
| L1 observations | 英文 observation、实体、动作、状态、confidence | 按视频建立，供检索和核验 |
| L2 events | 按实体/动作/相邻时间聚合的事件和状态变化 | 合并重复观察，保留冲突版本 |
| L3 working set | 当前问题的少量 memory ID 与工具结果 | 问题结束后清空，避免跨问题污染 |

每条记忆带 `video_id` 命名空间和 provenance。相邻 chunk 的高相似观察可合并；冲突观察不
覆盖，而是进入 `quarantined`。状态机为
`candidate -> verified`、`candidate -> quarantined` 或 `candidate -> stale`。

### 3.3 工具与核验门控

推理角色只能生成白名单 JSON。`search_memory(query)` 以 query 与 observation 的词元、实体、
动作重叠进行可解释召回，并加入 confidence、访问次数和 verified 奖励；它只产生候选，不会
直接改变最终答案。`inspect_segment(start_time,end_time,target)` 回取模型指定的局部时间段，
生成新的 candidate observation。`compare_segments` 对两个时间段进行状态差异判断。

`verify_evidence(memory_ids,target)` 根据 L0 指针重新读取原始帧，由感知角色输出 `SUPPORTED`、
`UNSUPPORTED` 或 `CONFLICT`：只有 `SUPPORTED` 才升级为 `VERIFIED`；`UNSUPPORTED` 保持候选并
禁止进入最终上下文；`CONFLICT` 进入隔离区，等待后续比较或不再使用。该门控把“语言上看似
合理的摘要”与“有原始视觉支持的事实”区分开来，降低错误记忆污染。

模型可自主决定是否继续观察：若当前工作集足以支持问题中的实体、时间关系和结论，则输出
`answer`；否则请求局部工具，最多执行 `max_actions` 次。工具只访问相关时间段，因此其优势
不是增加全视频抽帧，而是将视觉预算集中到可疑或高价值证据。

### 3.4 感知-推理解耦与 Hybrid-4

感知角色先建立问题无关的粗粒度索引，推理角色在查询时检索和核验。索引中的每条 observation
都能回指原始 chunk，压缩损失可以通过 `inspect_segment` 和 `verify_evidence` 局部修正。最终
上下文固定为：

```text
历史信息：VERIFIED evidence memory
当前信息：因果最近 4 帧原始视觉
```

因此 Hybrid-4 不是 Recent4 的别名：Recent4 只有当前帧，没有长期索引、provenance 和核验。

## 4. 实验设置与可复现性

### 4.1 数据与指标

使用公开 OVO-Bench，共 1,640 个问题：Backward 631、Realtime 837、Forward 172。Backward
按 EPM/HLD/ASI 取任务宏平均，Realtime 按 STU/OJR/ATR/FPD/ACR/OCR 取任务宏平均；本文主表
报告 Backward 和 Realtime，`Total=(Backward+Realtime)/2`。Forward 由独立入口补充评测，不
并入主表 Total。

### 4.2 统一配置

- 基座：冻结 `Qwen3-VL-2B/8B-Instruct`，不训练、不微调；
- 视频读取：`chunk_duration=1.0`、`fps=1.0`；
- VeriStream：`coarse_stride=12`、`max_frames_per_observation=8`、`max_actions=4`、
  `final_recent_frames=4`；
- NCCL 多卡运行设置 `NCCL_P2P_DISABLE=1 NCCL_IB_DISABLE=1`；
- 生成 prompt 使用英文，源代码注释使用中文；结果目录按实验名隔离，可从 rank checkpoint 断点续跑。

### 4.3 主结果（Backward/Realtime）

| 模型 / 方法 | Backward | Realtime | Total (B/R) |
| --- | ---: | ---: | ---: |
| Qwen3-VL-2B Recent4 | 53.56 | 74.23 | 63.90 |
| Qwen3-VL-2B Recent4 + CLIP-TopK16 | 52.73 | 69.52 | 61.13 |
| Qwen3-VL-2B Action-fact + Uniform16 | 56.91 | 73.68 | 65.30 |
| Qwen3-VL-2B Action-fact + CLIP-TopK16 | 57.30 | 73.59 | 65.45 |
| Qwen3-VL-2B State-aware + Uniform16 | 56.34 | 74.32 | 65.33 |
| Qwen3-VL-2B State-aware + CLIP-TopK16 | 54.11 | 73.32 | 63.72 |
| Qwen3-VL-8B Recent4 | 53.92 | 81.47 | 67.70 |
| Qwen3-VL-8B Recent16 | 55.06 | 77.80 | 66.43 |
| Qwen3-VL-8B Recent32 | 57.35 | 76.57 | 66.96 |
| Qwen3-VL-8B Action-fact + Uniform16 | 62.09 | 79.65 | 70.87 |
| Qwen3-VL-8B Action-fact + CLIP-TopK16 | 61.93 | 78.36 | 70.15 |
| VeriStream text-only DualRole (8B) | 55.05 | 44.91 | 49.98 |
| **VeriStream Hybrid-4 (8B)** | **57.60** | **79.18** | **68.39** |

Hybrid-4 相比 Recent4 的 Backward 提升 3.68 个百分点，Total 提升 0.69 个百分点；相比
text-only DualRole，Realtime 提升 34.27 个百分点。其主要收益来自 HLD（61.83%，相对
Recent4 的 52.15% 提升 9.68 个百分点），说明带 provenance 的历史证据对长期事件和位置问题
有效。ACR 仍比 baseline 低 6.42 个百分点，表明细粒度空间关系仍是瓶颈。

### 4.4 Forward 补充结果

Hybrid-4 Forward 的 REC/SSR/CRR 分别为 **20.34/54.37/42.50**，Forward 宏平均 **39.07**。
Forward 使用独立入口和相同记忆配置，因而不与上面的 Backward/Realtime Total 混算。

## 5. 消融与失败分析

### 5.1 为什么扩大窗口不是充分方案

Recent32 相比 Recent4 在 8B 上 Backward +3.43，但 Realtime -4.90；Uniform32 和 CLIP-TopK32
的 Total 进一步降至 59.10 和 57.80。均匀或语义抽帧仍把大量无关视觉 token 一次性放入上下文，
无法保证模型找到因果关键帧。

### 5.2 为什么 text-only DualRole 失败

text-only 的最终推理不接收原始视觉，导致 OCR、左右关系、小目标外观和瞬时动作无法从摘要中
恢复；严格核验门控还会在摘要不充分时拒答。其 Realtime 仅 44.91%，说明感知与推理解耦本身
不是充分条件，必须保留当前原始视觉通道。

### 5.3 Hybrid-4 的边界

Hybrid-4 恢复了 Realtime，但没有在所有子任务上超过 Recent4：STU/OJR/FPD/ACR/OCR 仍有不同
程度下降。当前检索是词元级而非 embedding 级，同义表达可能漏召回；每个问题重复建立索引也
带来较高延迟。工程日志还记录过 decord EOF 和 ffmpeg 资源错误，数据读取失败应与模型错误
分开统计，恢复运行时需检查 jsonl checkpoint 完整性和磁盘空间。

### 5.4 借鉴与差异

我们借鉴 VideoDeepResearch 的主动工具调用、VideoLucy 的可回溯记忆思想和 MemDreamer 的感知-
推理解耦视角，但未调用任何现成视频 agent 框架。自身设计差异为：冻结 Qwen3-VL；自主实现
JSON 工具白名单、L0-L3 memory 状态机和时间 provenance；核验通过后才允许证据进入答案上下文；
并显式采用 VERIFIED 历史文本 + 最近 4 帧原始视觉的 Hybrid-4 结构。

并使用独立结果目录。数据下载、环境配置和完整参数说明见 [README](README.md)。

## 7. 结论

预实验建立了清晰的机制链条：增加原始窗口会引入历史信息但损害当前聚焦；文本记忆恢复部分
长期能力但丢失不可压缩的视觉细节；可回指、可核验的证据记忆进一步降低错误摘要风险，而
最近 4 帧保留当前状态。VeriStream Hybrid-4 在 OVO-Bench 上验证了这一折中，但 ACR、语义检索
和运行效率仍限制其上限。后续工作应优先进行按视频索引缓存、embedding/词元混合检索、区域级
观察和 Hybrid-8 预算实验，而不是继续无差别扩大输入帧数。
